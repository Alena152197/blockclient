# Инструкция по копированию файла в Strapi

## Шаг 1: Подключитесь к серверу Strapi

Подключитесь к серверу, где установлен Strapi (`https://324cbb377ef9.vps.myjino.ru`)

## Шаг 2: Найдите папку с моделью Post

Найдите папку:
```
src/api/post/content-types/post/
```

Если структура другая, поищите:
- `api/post/`
- `content-types/post/`
- Любую папку с файлами модели Post

## Шаг 3: Создайте файл lifecycles.js

В папке `src/api/post/content-types/post/` создайте файл `lifecycles.js`

## Шаг 4: Скопируйте содержимое

Откройте файл `strapi-lifecycle-hook.js` из этого проекта и скопируйте всё его содержимое в созданный файл `lifecycles.js` на сервере Strapi.

## Шаг 5: Перезапустите Strapi

После создания файла перезапустите сервер Strapi:
```bash
npm run develop
# или
npm run start
```

## Шаг 6: Проверьте работу

1. Создайте пост через консоль в вашем приложении
2. Проверьте логи Strapi - должно появиться сообщение: `✅ Автоматически установлен автор для поста: ...`
3. Проверьте, что пост появился в "Мои интересы"

## Альтернатива: Если нет доступа к серверу

Если у вас нет прямого доступа к серверу Strapi, попросите администратора сервера:
1. Создать файл `src/api/post/content-types/post/lifecycles.js`
2. Скопировать содержимое из файла `strapi-lifecycle-hook.js`
3. Перезапустить Strapi

## Что делает этот файл?

Файл `lifecycles.js` автоматически устанавливает автора поста из токена пользователя при создании поста через API. Это решает проблему, когда поле `author` нельзя установить напрямую через API из-за ограничений прав доступа.
